GABARITO: RESULTADOS ESPERADOS

1. Caso 1: FIFO vs. SJF (Sem Sobrecarga)

   Objetivo: Validar FIFO e SJF (não-preemptivo) com sobrecarga_contexto zerada.

   Resultado Esperado (FIFO/FCFS):
     Ordem de Execução: P1 -> P2 -> P3 (seguindo a ordem de chegada ou ID, já que todos chegam em t=0).
     Tempos de Término: P1 em t=5, P2 em t=7, P3 em t=10.

   Resultado Esperado (SJF Não-Preemptivo):
     Ordem de Execução: P2 -> P3 -> P1 (baseado no menor tempo de 'execucao' total).
     Tempos de Término: P2 em t=2, P3 em t=5, P1 em t=10.

2. Caso 2: "Caso Base" Round-Robin (Com Sobrecarga)

   Objetivo: Validar o Round-Robin, quantum, sobrecarga_contexto e chegadas em tempos diferentes.

   Resultado Esperado (RR, Quantum=2, Sobrecarga=1):
     t=0-2: P1 executa (restam 3 u.t.).
     t=2-3: Sobrecarga.
     t=3-5: P2 executa (P2 chegou em t=1, P3 chegou em t=3. Fila de prontos: [P2, P3, P1]) (restam 2 u.t.).
     t=5-6: Sobrecarga.
     t=6-8: P3 executa (Fila: [P3, P1, P2]) (restam 0 u.t.) -> P3 TERMINA.
     t=8-9: Sobrecarga. (Nota: P1 estoura seu deadline de t=8 neste momento).
     t=9-11: P1 executa (Fila: [P1, P2]) (resta 1 u.t.).
     t=11-12: Sobrecarga.
     t=12-14: P2 executa (Fila: [P2, P1]) (restam 0 u.t.) -> P2 TERMINA.
     t=14-15: Sobrecarga.
     t=15-16: P1 executa (Fila: [P1]) (resta 0 u.t.) -> P1 TERMINA.
     Na tabela final: P1 deve ter 'deadline_ok? = Nao'.

3. Caso 3: Preempção EDF

   Objetivo: Validar que o EDF é preemptivo quando um processo mais urgente chega.

   Resultado Esperado (EDF, Sobrecarga=1):
     t=0-2: P-Longo (deadline 20) executa.
     t=2: P-Urgente (deadline 8) chega. Como 8 < 20, ocorre Preempção.
     t=2-3: Sobrecarga (devido à preempção).
     t=3-6: P-Urgente executa suas 3 u.t. -> P-Urgente TERMINA (antes do seu deadline 8).
     t=6-7: Sobrecarga (para P-Longo voltar à CPU).
     t=7-15: P-Longo retoma e executa suas 8 u.t. restantes -> P-Longo TERMINA.

4. Caso 4: CPU Ociosa

   Objetivo: Validar a Regra Geral 4, onde o tempo deve avançar se a fila de prontos estiver vazia.

   Resultado Esperado (Qualquer algoritmo):
     t=0-3: P1 executa -> P1 TERMINA.
     t=3-8: CPU OCIOSA (não há processos prontos).
     t=8-10: P2 chega (em t=8) e executa -> P2 TERMINA.
     Métrica Chave: O relatório final deve mostrar 5 unidades de tempo ocioso (de t=3 a t=8).

5. Caso 5: Justiça CFS-Sim

   Objetivo: Validar o cálculo do vruntime e a influência da prioridade no CFS-Sim.

   Cálculos de Peso:
     P1-PriAlta (p=1): w = 1.25^(1-1) = 1.0
     P2-PriBaixa (p=3): w = 1.25^(3-1) = 1.5625

   Resultado Esperado (CFS-Sim, Sobrecarga=0):
     O vruntime de P2-PriBaixa crescerá 1.56x mais rápido que o de P1-PriAlta sempre que executar.
     Para manter os vruntimes equilibrados (o objetivo do CFS), o escalonador deve dar mais fatias de CPU para P1-PriAlta.
     Traço (exemplo, dt=1):
       t=0: vruntimes = {P1: 0, P2: 0}. Escolhe P1.
       t=1: vruntimes = {P1: 1.0, P2: 0}. Escolhe P2.
       t=2: vruntimes = {P1: 1.0, P2: 1.56}. Escolhe P1.
       t=3: vruntimes = {P1: 2.0, P2: 1.56}. Escolhe P2.
       t=4: vruntimes = {P1: 2.0, P2: 3.12}. Escolhe P1.
       t=5: vruntimes = {P1: 3.0, P2: 3.12}. Escolhe P1.
     O Gantt deve mostrar P1 executando com mais frequência que P2.

6. Caso 6: Bônus de Memória (Page Fault)

   Objetivo: Validar o bloqueio por E/S (disco) quando um page fault ocorre, usando o custo_disco.

   Evento Simulado: Este teste assume que seu código irá forçar um Page Fault em P1 no instante t=2.

   Resultado Esperado (Ex: RR, Custo Disco=3, Sobrecarga=1):
     t=0-2: P1 executa.
     t=2: P1 sofre PAGE FAULT. P1 é movido para a fila de "Bloqueados" por 3 u.t. (até t=5). A CPU fica livre.
     t=2-3: Sobrecarga (para P2, que já estava pronto, assumir).
     t=3-5: P2 executa (chegou em t=1) -> P2 TERMINA.
     t=5: P1 termina o bloqueio de disco e volta para a fila de prontos. A CPU executa P1.
     t=7: P1 executa suas 2 u.t. restantes -> P1 TERMINA.
     Gantt: O gráfico deve mostrar P1 em execução (verde), depois bloqueado (azul claro) de t=2 a t=5, enquanto P2 executa nesse intervalo.
